# Claude zh-CN Manager 实现计划

## P0：登录回调修复

目标：让 Google 登录完成后回到同一个汉化版窗口。

任务：

1. 读取当前 `claude://` 协议处理器。
2. 备份当前协议处理器到 `backups/oauth-protocol/`。
3. 调用补丁工具的 `--prepare-oauth-login`，让回调指向汉化启动器。
4. 验证启动器是否带 `--user-data-dir`。
5. 诊断报告中显示当前回调状态。

验收：

- `claude://` 指向 `launch_claude_zh_cn.vbs`。
- 登录回调进入 `ClaudeZhCN-3p` 用户数据目录。

## P1：一键更新并重新汉化

目标：官方更新后，通过一个脚本完成新版同步和汉化重建。

任务：

1. 调用补丁工具 `--check-update` 获取版本状态。
2. 调用补丁工具 `--force-download` 下载并重建新版汉化版。
3. 合并本地增量翻译到最终 `zh-CN.json`。
4. 调用补丁工具 `--apply-user-settings` 重建启动器和快捷方式。
5. 输出汉化覆盖率和英文回退数量。

验收：

- 能重新生成汉化版 Claude。
- 本地增量翻译不会被基础翻译覆盖。
- 更新失败时不删除旧版本。

## P2：菜单化管理器

目标：提供一个用户能直接运行的入口。

菜单：

1. 启动 Claude zh-CN。
2. 诊断当前状态。
3. 修复登录回调。
4. 检查官方更新。
5. 一键更新并重新汉化。
6. 应用本地增量翻译。
7. 恢复 OAuth 回调备份。

验收：

- 不需要记命令。
- 每个动作执行前能说明影响。
- 危险动作不自动执行删除。
